# Version Badge

**Date:** 2026-03-11

## Overview

Added an unobtrusive build version badge to the application header so the running
version can be validated in production without requiring log access.

## Files Modified

| File | Change |
|------|--------|
| `build.rs` | **New** — build script that captures the short git SHA via `git rev-parse --short HEAD` and exposes it as the `GIT_SHA` compile-time env var. Falls back to `"unknown"` when git is unavailable. |
| `src/app.rs` | Added `<span class="app-version">` inside `.header-left`, displaying `v{CARGO_PKG_VERSION}+{GIT_SHA}` using `concat!()` for zero-runtime cost. |
| `style/main.scss` | Added `.app-version` rule — monospace font, 0.65 rem, muted colour at 55% opacity, `user-select: all` for easy copy-paste. |

## Design Decisions

- **Compile-time only** — both the cargo version and git SHA are embedded at build
  time via `env!()` macros; no runtime overhead or additional server function.
- **`user-select: all`** — lets the operator triple-click the badge to copy the full
  version string for pasting into issue reports.
- **Placement** — inline with the title in `.header-left`, appearing after the live
  badge.  Low visual weight (muted colour + reduced opacity) so it doesn't compete
  with the accent-coloured title.
- **`cargo:rerun-if-changed`** targets `.git/HEAD` and `.git/refs/` so the binary is
  re-stamped on every commit or branch switch without unnecessary full rebuilds.
