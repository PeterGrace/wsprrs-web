# 2026-03-10T01:00:00 — UX Fixes

## Changes

### Relative time-window selector (`src/components/filter_panel.rs`)

Replaced the two `<input type="datetime-local">` (From / To UTC) controls
with a single `<select>` offering predefined relative windows:

| Label    | Seconds back |
|----------|-------------|
| 1 hour   | 3 600       |
| 2 hours  | 7 200       |
| 4 hours  | 14 400      |
| 8 hours  | 28 800      |
| 12 hours | 43 200      |
| 24 hours | 86 400      |
| 48 hours | 172 800     |
| 7 days   | 604 800     |

Selecting an option writes `filter.since_unix = Some(now - secs)` and
clears `filter.until_unix`.  A local `RwSignal<i64>` (`window_secs`)
tracks the current selection so the Reset button can restore it to
1 hour alongside resetting the `SpotFilter`.

Removed the now-unused `unix_to_datetime_local` and `datetime_local_to_unix`
helpers.

### Live update completeness (`src/app.rs`)

The `on_spots` SSE callback previously only called
`map_spots_resource.refetch()`.  Added `spots_resource.refetch()`,
`stats_resource.refetch()`, and `bands_resource.refetch()` so the spot
table, stats bar, and band counts all update when a live event arrives.

### Leaflet SRI removed (`src/app.rs`)

Removed `integrity` and `crossorigin` attributes from the Leaflet 1.9.4
CDN `<link>` and `<script>` tags.  SRI hashes block resource loading when
a CDN edge node or proxy modifies the response in any way.

### Map bridge diagnostics (`src/components/map.rs`)

Added `web_sys::console::error_1/2` logging in the JS bridge functions
(`call_js_init_map`, `call_js_update_map`, `call_js_highlight_grid`) so
a missing or non-functional `window.wsprMap` produces a visible console
error instead of silently returning.  Extracted shared helper
`js_wspr_map()` to avoid repeating the lookup and null-check.

### map.js load confirmation (`public/map.js`)

Added `console.log("[wsprMap] map.js loaded")` at script startup so it is
easy to confirm in DevTools that the file is being served and executed.
